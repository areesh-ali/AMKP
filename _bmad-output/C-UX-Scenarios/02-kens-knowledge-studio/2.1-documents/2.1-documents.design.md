## Overview

**Page Purpose:** Browse and upload Documents for the Active Tenant; see parse status progress.

**Success Criteria:** Upload appears in list; status progresses to parsed; wrong Tenant never shown.

**Entry Points:** Left rail Documents · after Maya standup

**Exit Points:** Document detail row click

## Layout Structure

```
+----------+----------------------------------+
| rail     | Documents · Tenant chip          |
|          | [Upload] [Filter status]         |
|          | table: name · status · version   |
+----------+----------------------------------+
```

## Sections

### Toolbar
- Primary Upload (teal)
- Filter chips: all / pending / parsed / failed
- Count muted

### Table
- Dense rows; status pills with text labels
- Mono version; relative updated time
- Row click → detail

## Storyboard states

| State | Treatment |
| --- | --- |
| Empty | Fraunces empty + Upload CTA |
| Uploading | Optimistic row pending |
| Parsed | ok pill |
| Failed | danger pill + reparse hint |
| Cross-Tenant | never list other Tenant docs |

## Content (EN)

- Title: Documents
- Upload: Upload
- Empty: No documents yet — upload to ground Retrieve.

- listDocuments(tenant)
