## Overview

**Page Purpose:** Let Ken ask the plane with uploads attached, watch the agent work, and land on Evidence — not a free-form final answer.

**Success Criteria:** Query submitted; working steps visible; stream advances to Evidence climax with cost; Active Tenant always shown.

**Entry Points:** Left rail · Document detail “Ask” · Onboarding deep-link  
**Exit Points:** Thread/Retrieve detail (on complete) · Trace · Documents

## Sections

### 1. Left rail
- AMKP wordmark (`type-display` ~22px)  
- Nav: Studio (active), Documents, Traces, Eval, Policy  
- Recent threads list (title + relative time)  

### 2. Top bar
- Thread title (editable later; v1 auto from first query)  
- **TenantChip** `support` (click → switcher; confirm if dirty)  
- Health dot  

### 3. Stream (empty → working)
- Empty: Fraunces line “Ask the plane — Evidence, not vibes.” + short sub  
- After submit: user bubble (soft elevated, no chatty avatar required)  
- **WorkingStep** rows stagger in (`color-working` label)  

### 4. Composer
- Textarea placeholder: “Ask about your Tenant knowledge…”  
- Attach control (paperclip + drag onto composer)  
- PreferCorrectness checkbox (default on)  
- Mode: single_pass | agentic  
- Primary: teal “Run” · shortcut ⌘Enter  

---

## Storyboard states

| State | Treatment |
| --- | --- |
| Empty | Display empty + composer focus |
| Submitting | Composer disabled; working row “Routing…” |
| Working hops | Expandable steps from Trace preview |
| Complete | Navigate/inline promote to 2.4 Evidence |
| Error / deny | Inline “Not available for this Tenant” — no bleed |
| Plane down | Top banner; mutate disabled |
