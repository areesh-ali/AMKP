AMKP Architecture Walkthrough

For eng team kickoff. Spine is canonical — this page is the tour.

1. Paradigm in one line

Hexagonal modular monolith: REST and MCP are adapters over the same application services. Domain never talks to Postgres directly. Two processes: api + worker.

2. Decisions that stop epic drift

AD-2 / AD-3 — Tenant only from auth; fail-closed namespace per Tenant; cache keys include tenant_id.
AD-4 — Retrieve returns EvidenceEnvelope only (no generate API in MVP).
AD-5 / AD-10 — Heavy parse/eval on BullMQ workers; API stays thin.
AD-6 — MCP = thin Streamable HTTP facade; same services as REST.
AD-7 / AD-8 — OTel everywhere agentic; default single-pass; max hops 3.
AD-9 — Postgres + pgvector is SoR + vectors for MVP.

3. Cold-start tree

apps/api  apps/worker
packages/domain  application  adapters-*  sdk-js
infra/docker-compose.yml

4. Stack (seed — code owns later)

  • Node 24 LTS · NestJS 11 · TypeScript · Prisma
  • PostgreSQL 16 + pgvector · Redis 7 · BullMQ
  • OpenTelemetry · MCP SDK

5. Build order (from tickets)

S1: tenancy + namespace isolation + ingest stub → S2: retrieve + TableEvidence → S3: PreferCorrectness + Leak CI + traces → then agentic / eval / SDK.
