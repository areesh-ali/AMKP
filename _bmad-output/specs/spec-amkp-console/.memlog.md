- (direction) User chose wedge C: both Platform Admin console and Developer workspace; explicitly not an MVP — treat as a full product.
- (decision) Product sits ON TOP of AMKP plane (API/SDK/MCP); does not replace or become Glean-class workforce search.
- (decision) Slug/product working name: AMKP Console (spec folder spec-amkp-console).
- (capability) CAP-1: Session auth — Platform Admin and Tenant Operator roles can sign in and act only within authorized scope.
- (capability) CAP-2: Tenancy ops — Admin can create/list Accounts, Tenants, issue/revoke/rotate API keys, view audit.
- (capability) CAP-3: Knowledge workspace — Operator can ingest/upload Documents, browse versions/status/chunks, reparse/delete/prune, download content.
- (capability) CAP-4: Retrieve studio — Operator can run single-pass/agentic Retrieve, inspect Evidence with citations, PreferCorrectness, CostEstimate.
- (capability) CAP-5: Trace & cost inspector — Operator can open Trace by requestId and see router, hops, evidence IDs, costs; no cross-Tenant leak.
- (capability) CAP-6: Eval console — Operator can run golden-set and TableRank evals, view reports, access POC Pack entry points.
- (capability) CAP-7: Tenant policy controls — Operator can view/edit Tenant flags (pageVision, agentic readiness/override, PreferCorrectness threshold) with audit.
- (capability) CAP-8: Ops health — Admin can see plane health/ready/adapters, metrics summary, optional storage orphan dry-run sweep.
- (capability) CAP-9: DX onboarding — New developer can complete signup→key→ingest→retrieve guided path in-product in under 60 minutes.
- (constraint) Console consumes AMKP only via public HTTP/SDK contracts; no direct DB access from the UI process.
- (constraint) Tenant isolation fail-closed in UI: never show or request other Tenants' data; session Tenant binding mandatory.
- (constraint) Primary Retrieve UX presents Evidence + citations + cost — not a final LLM answer as the product contract.
- (constraint) Product-quality bar: production auth, durable sessions, accessible UI, error states, empty states, loading, and auditability — not a demo shell.
- (assumption) Working product name AMKP Console until branding decided.
- (assumption) Plane AMKP SPEC-amkp remains the backend contract; Console SPEC adopts it and does not fork CAP semantics.
- (assumption) Web app (SPA or SSR) in the monorepo as apps/console; English-first UI.
