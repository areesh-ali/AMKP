UJ-1

Maya stands up a second Product Tenant in a day

Persona: Maya — platform engineer at a mid-market SaaS. Support and Docs bots are forking the same RAG stack. She needs one shared knowledge plane for both products.
Entry

Account setup

Creates Account via console; provisions API keys for two Tenants: support and docs.

Ingest

Corpus upload

Uploads PDF corpus to each Tenant via Ingest API. SDK sample from docs accelerates first call.

Retrieve

Evidence check

Calls Retrieve on both Tenants. Responses return citation-obligatory Evidence objects.

Integrate

MCP wiring

Connects MCP retrieve tool into both Product agents (Cursor / Claude Desktop config).

Climax

Isolation verified

Both Products return Evidence with citations. Leak-test job from docs shows zero cross-Tenant hits.

Edge case

Parse escalation

Scanned deck triggers Parse Ladder escalation. Maya sees parse_confidence on returned Evidence.

Sally UX Notes

Jobs to be done
  • Functional: Ship grounded AI across ≥2 Products without a three-person ML platform team.
  • Functional: Prove Tenant A never retrieves Tenant B's Evidence.
  • Social: Become the internal platform other squads adopt instead of forking LangChain apps.
Emotional outcomes
  • Confidence that tenancy is hard, not filter-based.
  • Relief from duplicated RAG maintenance across products.
  • Pride in shipping both bots in one day from docs alone.

POC Pack

UJ-3

Priya debugs a bad answer without guessing

Persona: Priya — product engineer on Support. A customer-facing bot returned a wrong answer; she needs root cause, not vibes.
Entry

Wrong answer

Support bot returns incorrect policy detail. Priya captures request ID from Product logs.

Trace

Open trace

Fetches Trace via API/SDK. Sees retrieved Evidence IDs, parse path, router decision, CostEstimate.

Diagnose

Root cause

Identifies stale Document version in index. Trace shows old version ID on returned Evidence.

Fix

Re-ingest

Re-ingests updated Document. Confirms new version watermark on subsequent Retrieve.

Climax

Regression caught

Regression eval catches the case. Golden-set fixture now passes for the corrected policy question.

Edge case

PreferCorrectness

On a low-coverage query, PreferCorrectness mode would have refused instead of guessing.

Sally UX Notes

Jobs to be done
  • Functional: Debug retrieval failures with full observability—not log archaeology.
  • Functional: Fix stale content and prevent recurrence via eval regression.
  • Functional: Understand router choice (single-pass vs agentic) per request.
Emotional outcomes
  • Agency: every bad answer has a traceable chain of Evidence.
  • Reduced anxiety about shipping AI features to customers.
  • Speed: minutes to root cause, not days of guesswork.
