AMKP Product Overview

STAKEHOLDER NARRATIVE · RAG-SOL · 2026-07-14

The Problem

Platform teams must ship grounded AI across several products. They stitch vector DBs, parsers, frameworks, and ad-hoc ACL filters—and get inconsistent quality, duplicated spend, cross-tenant risk, and agent demos that become proofs of cost.

The missing piece is not "an LLM." It is a shared plane that combines multimodal evidence fidelity, hard tenancy, and controlled multi-hop—integrable behind many products.

Who this serves: Product and platform engineers at multi-product software companies and ISVs. Success means a new product tenant live in about a day, zero cross-tenant leaks in soak testing, and AI features shipped without a three-person ML platform hire.

The Solution

AMKP exposes ingest and retrieve as first-class APIs (plus MCP tools). Customers get evidence objects with citation-obligatory, typed payloads where possible; silo/namespace-default multi-product tenancy with JWT-derived identity and automated leak tests; a structure-aware multimodal parse ladder; and a guarded agentic router that defaults to single-pass with capped multi-hop only when needed.

BYO LLM is supported. AMKP owns grounding infrastructure, not the chat surface.

Four Pillars

PILLAR 01

Hard Tenancy

Silo/namespace-default multi-product isolation. Tenant derived from auth—never from client-supplied free text.

  • JWT / API key scoped credentials
  • Tenant-keyed caches
  • Automated leak-test CI
PILLAR 02

Multimodal Structure

Parse Ladder: cheap extract → layout-aware → optional VLM/page-vision. TableEvidence preserves headers and cells.

  • TableRank as hero quality metric
  • parse_confidence on every chunk
  • Tables and charts, not just prose
PILLAR 03

Guarded Agentic

Router defaults to single-pass. Multi-hop only when needed—with hop budgets, circuit breakers, and full traces.

  • Agentic Readiness gate
  • Cost attribution per request
  • PreferCorrectness over guessing
PILLAR 04

Eval / POC Pack

Sealed bake-off kit: golden set, ACL adversarial suite, multimodal chart pack, and cost simulator.

  • Self-serve from public docs
  • Faithfulness sampling
  • Deterministic fixture scores

Non-Goals

  • Glean-class employee search UI
  • Unbounded agentic-by-default loops
  • Training on customer data
  • Competing as a general vector DB
  • Full connector marketplace (API/MCP first)
  • Air-gapped appliance / VPC as MVP default
  • Primary business of seat-based chat to knowledge workers

Success Metrics

TARGET 01
Time-to-first-retrieve from docs < 60 minutes on free/self-serve path.
TARGET 02
Cross-tenant leak tests: zero failures in continuous soak testing for design partners.
TARGET 03
Multimodal gold set: material lift on table/chart questions vs text-only baseline.
TARGET 04
≥90% of production queries served via single-pass retrieval; agentic hops within budget caps.
TARGET 05
Design partners: ≥2 products on one plane within the first expansion cycle.
